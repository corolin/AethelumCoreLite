## 问题分析

测试失败的原因是路由器在激活时没有创建强制性队列，导致后续注册钩子时失败。具体来说：

1. 路由器类中定义了`MANDATORY_QUEUES`集合，包含了`Q_PROCESS_INPUT`、`Q_RESPONSE_SINK`、`Q_ERROR_HANDLER`和`Q_DONE`四个强制性队列
2. 但在`activate`方法中，只是设置了激活状态，并没有实际创建这些队列
3. 当`auto_setup`方法调用`_setup_error_handler()`时，会尝试向`Q_ERROR_HANDLER`队列注册钩子
4. 由于`Q_ERROR_HANDLER`队列尚未创建，`register_hook`方法会抛出`ValueError: 队列 Q_ERROR_HANDLER 不存在，请先创建队列`

## 修复方案

修改`NeuralSomaRouter`类的`activate`方法，在激活系统时自动创建所有强制性队列：

1. 在`activate`方法中，遍历`MANDATORY_QUEUES`集合
2. 为每个队列调用`create_queue`方法创建队列实例
3. 确保在调用任何需要队列的方法之前，所有强制性队列都已创建

## 修复步骤

1. 打开`aethelum_core_lite/core/router.py`文件
2. 修改`activate`方法，添加创建强制性队列的逻辑
3. 运行测试脚本验证修复效果

# 在activate方法中添加队列创建逻辑
def activate(self) -> None:
    """
    激活整个神经系统

    自动创建所有强制性队列，启动必要的工作器。
    """
    with self._lock:
        if self._router_active:
            self.logger.warning("神经系统已经激活")
            return
        
        # 创建所有强制性队列
        for queue_name in self.MANDATORY_QUEUES:
            self.create_queue(queue_name)
        
        self._router_active = True
        self._stats['activated_at'] = time.time()
        self.logger.info("神经系统已激活")
